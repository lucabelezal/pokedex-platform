# Padrões De Projeto

Esta seção reúne referências de design patterns e notas práticas para Go no contexto deste projeto.

## O Que Vamos Ver

- Quais design patterns valem a pena conhecer em Go.
- Quais patterns se encaixam naturalmente com arquitetura hexagonal.
- Quais patterns são mais relevantes para este repositório.
- Onde encontrar exemplos canônicos em Go para cada pattern.

## Recapitulação

O objetivo desta seção não é forçar patterns na base de código. O objetivo é construir repertório sobre patterns, reconhecer estruturas úteis quando elas aparecem naturalmente e evitar complexidade acidental quando um design mais simples já é suficiente.
